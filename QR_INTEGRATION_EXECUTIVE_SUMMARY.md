## How It Works End-to-End

### Employee Workflow
```
1. Open Dashboard
2. Scroll to "Time In/Out" section
3. Click blue "📱 QR Scanner" button
4. Allow camera access
5. Point phone camera at QR code (generated by HR)
6. System auto-detects time in OR time out
7. Shows green or orange confirmation modal
8. Click OK to confirm
9. Time recorded in database
10. Back to scanner ready for next scan
```

### HR Admin Workflow
```
1. Open Dashboard (as HR Admin)
2. Scroll to "Time In/Out" section
3. Click green "🟢 Generate QR" button (appears only for HR)
4. Enter how many QR codes (1-50)
5. Enter server IP if needed
6. Click "Generate QR Codes"
7. Choose: Print, Download, or Share
8. Display on office screen OR print for bulletin board
9. Employees scan within 1 minute
10. Tokens auto-expire after use or 1 minute
```

## Database Integration

### Tables Used
```
attendance_tokens   ← Stores QR tokens
├─ token (unique code)
├─ created_at (timestamp)
├─ expires_at (1-minute validity)
├─ used (boolean)
└─ ip_address (server IP)

attendance         ← Time records
├─ employee_id
├─ time_in
├─ time_out
├─ status
└─ total_hours_worked

employees          ← Employee data
users              ← User accounts  
audit_log          ← Action tracking
```

### Token Lifecycle
```
09:00:00 - Admin generates token → INSERT into attendance_tokens
09:00:30 - Employee scans → VALIDATE token (not used, not expired)
09:00:45 - Time recorded → UPDATE token as used
09:02:00 - Cleanup runs → DELETE expired tokens
```

## Security Features Implemented

✅ **Token Security**
- Cryptographic generation (`random_bytes(32)`)
- Base64 encoding for storage
- 1-minute automatic expiry
- Single-use enforcement

✅ **Access Control**
- HR Admin role required for generation
- Authentication check required
- Role-based conditional display

✅ **Data Protection**
- Prepared statements (SQL injection prevention)
- HTML escaping (XSS prevention)
- CSRF tokens where applicable
- Audit logging of all actions

✅ **Error Handling**
- Invalid token detection
- Expired token cleanup
- User-friendly error messages
- Database transaction rollback on failure
